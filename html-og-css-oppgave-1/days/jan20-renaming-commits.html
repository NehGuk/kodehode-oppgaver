Renaming previous commits

First things first: always make sure to write clear and consistent commit messages. However, things may go wrong at times. Silly typos, meaningless messages, inconsistent verb tenses. There's a lot we may want to improve. Today I played around with the infamous git rebase -i HEAD~N strategy. What the heck is that? This allows us to edit commit messages that have already been pushed to our repository. This is how it works: 


                git rebase -i HEAD~3

This will allow you to edit the last 3 commits. Use any number you wish here, as long as it's equal or smaller than the total number of commits of your project. The system will bring up a file with all the commit messages: for the ones you wanna change, just replace pick by reword. Then save it and close the document. A new file will open up, where you're finally able to edit the commit message. Again, save and close. Repeat the process for every commit message you wish to edit. Once you're done, proceed with the final trick: 


                git push --force origin

That's basically it. More details on the official docs here. An important note: it's best to avoid doing this in projects where several developers are working on the same repo. A poorly planned rebase will almost certainly be a major headache for the whole team –– because everything will go out of sync, and everyone in the team will hate you.
